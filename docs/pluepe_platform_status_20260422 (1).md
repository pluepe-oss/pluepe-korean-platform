|---|---|
| 이메일 + 비밀번호 | ✅ 구현 완료 |

- student → `/learn`
- admin → `/admin`
- master → `/master`

## 3. TOPIK1 모의고사 IBT 플레이어

### 화면 구조
```
/learn/exam             → 시험 허브 (TOPIK1/2, EPS-TOPIK 3카드)
/learn/exam/[type]      → IBT 플레이어
/learn/exam/result/[id] → 결과 페이지
```

### 현재 문항 현황

| 구분 | 문항 수 | 접근 |
|---|---|---|
| TOPIK1 읽기 | 5개 | 무료 |
| TOPIK1 읽기 | 3개 | 유료 |
| EPS-TOPIK 읽기 | 3개 | 무료 |
| TOPIK1 듣기 | 3개 | 무료 |
| **합계** | **14개** | 샘플 수준 |

### 실제 TOPIK1 시험과 비교

| 항목 | 실제 TOPIK1 | 현재 구현 |
|---|---|---|
| 듣기 문항 | 30문항 | 3개 (샘플) |
| 읽기 문항 | 40문항 | 8개 (샘플) |
| 쓰기 | 없음 (TOPIK1) | 없음 |
| 시간 제한 | 듣기 40분 / 읽기 60분 | 타이머 구현됨 ✅ |
| 문항 형식 | 4지선다 | 4지선다 ✅ |
| 채점 | 자동 | 자동 ✅ |
| 결과 분석 | 없음 | 카테고리별 취약점 분석 ✅ |

### 확장 가능성

✅ 구조 자체는 확장 설계 완료.
questions 테이블에 문항 추가만 하면 바로 반영됨.

```
exam_type: 'TOPIK1' / 'TOPIK2' / 'EPS-TOPIK'
category: 'listening' / 'reading'
is_free: true / false
```

### 비용 구조

| 항목 | 비용 |
|---|---|
| 플레이어 자체 | 없음 (자체 개발) |
| 문항 저장 | Supabase DB (현재 무료 플랜 범위) |
| 듣기 음원 | 현재 없음 — OpenAI TTS 또는 직접 녹음 필요 |
| AI 오답 해설 | Claude API 호출 시 비용 발생 |

### 핵심 문제점 (실서비스 전 해결 필요)

1. 문항 14개뿐 → 실제 시험 수준 문항 필요
2. 듣기 음원 없음 → TOPIK1 듣기 30문항 전부 음원 필요
3. 문항 교육적 검수 안 됨 → AI 생성 샘플 수준

### 실서비스 수준 달성 조건

- TOPIK1 기준 듣기 30 + 읽기 40 = **70문항 제작 필요**
- 듣기 음원 제작 필요
- 교육 전문가 검수 필요
