# TOPIK1 콘텐츠 라이브러리 PRD — 최종
> 작성일: 2026.04.23
> 핵심 원칙: "TOPIK1은 같은 문장을 많이 본 사람이 이기는 시험"

---

## 0. 설계 철학

> **"TOPIK1은 문장 암기 시험이 아니라, 표현 구조에 익숙해지는 시험이다."**
> **"핵심 어휘와 문형을 여러 상황에서 반복 노출받은 학습자가 잘 보는 시험이다."**

### 반복하면 안 되는 것 ❌
```
동일 문장      → 물 주세요. / 물 주세요. / 물 주세요.
동일 문제      → 같은 질문을 형태만 바꿔 반복
동일 정답 패턴 → 항상 ①번이 정답, 항상 같은 위치
→ 암기 피로만 생김. 시험에서 전혀 도움 안 됨.
```

### 반복해야 하는 것 ✅
```
핵심 문형      → NOUN + 주세요
핵심 어휘군    → 물/김밥/봉투/영수증/커피
핵심 의미 구조 → "무언가를 요청하는 표현"
핵심 상황 기능 → 편의점에서 물건 요청하기

올바른 반복 예시:
물 주세요.         ← 문형 1차 노출
삼각김밥 주세요.   ← 같은 문형 + 다른 어휘
봉투 주세요.       ← 같은 문형 + 다른 어휘
물 두 개 주세요.   ← 같은 문형 + 수량 추가
따뜻한 거 주세요.  ← 같은 문형 + 형용사 확장
→ NOUN + 주세요 구조를 몸에 익힘
```

### 반복의 방법
```
같은 문형을 → 다른 어휘와 수량 표현 속에서 → 7가지 형태로 노출
세션영상 → 단어카드 → 단어퀴즈 → 패턴 → 듣기 → 읽기 → 복습 → 테스트
```

### 핵심 원칙
```
모든 문항의 출처 = 영상 스크립트
스크립트의 핵심 문형을 다양한 어휘/상황으로 확장
스크립트 밖의 문형은 사용하지 않는다
```

---

## 1. 유닛 구조

```
1개 유닛 = 1개 스크립트 = 1개 상황
                ↓
        스크립트에서 추출
        ↓           ↓
    핵심 어휘     핵심 패턴
    (12~15개)    (5~8개)
        ↓           ↓
    문장 30~50개 생성
        ↓
    학습 흐름에 투입
```

---

## 2. 학습 흐름 (반복 구조)

```
① 세션 영상 (스크립트 전체 노출)
        ↓
② 단어 카드 (어휘 1차 노출)
        ↓
③ 단어 퀴즈 (어휘 2차 노출 — 문장 속에서)
        ↓
④ 패턴 학습 (문장 구조 인식)
        ↓
⑤ 패턴 퀴즈 (문장 변형 연습)
        ↓
⑥ 듣기 (같은 표현 음성으로 노출)
        ↓
⑦ 읽기 (같은 표현 문자로 노출)
        ↓
⑧ 복습 (오답 기반 재노출)
        ↓
⑨ 미니 테스트 (TOPIK 형식)
        ↓
⑩ AI 확장 (상황 변형)
```

### 4-1. 단어 퀴즈 (3가지 유형)

**[A] 상황 보고 단어 고르기**
```json
{
  "type": "situation",
  "question": "편의점에서 물건 값을 물어볼 때 쓰는 말은?",
  "options": ["얼마", "어디", "언제", "누구"],
  "answer": 0,
  "hint": "이거 ____예요?"
}
```

**[B] 뜻 설명 보고 단어 고르기**
```json
{
  "type": "meaning",
  "question": "돈을 내는 것을 뜻하는 말은?",
  "options": ["계산", "주문", "예약", "취소"],
  "answer": 0,
  "hint": "카드로 ____할게요."
```

**[C] 예문 빈칸 보고 단어 고르기**

## 5. 문장 뱅크 구조 (핵심)

```
스크립트 1개 → 핵심 문형 추출 → 문장 30~50개 확장

확장 원칙:
같은 문형 + 다른 어휘/수량/상황

예시 — 문형: NOUN + 주세요
① 물 주세요.
② 삼각김밥 주세요.
③ 봉투 주세요.
④ 영수증 주세요.
⑤ 물 하나 주세요.
⑥ 물 두 개 주세요.
⑦ 삼각김밥 하나 주세요.
⑧ 따뜻한 거 주세요.
⑨ 차가운 거 주세요.
⑩ 영수증은 괜찮아요.

→ 학습자는 문장을 외우는 게 아니라
→ NOUN + 주세요 구조를 몸에 익힌다

문장 뱅크 활용:
- 단어 퀴즈 예문으로 사용
- 패턴 퀴즈 선택지로 사용
- 듣기 스크립트로 사용
- 읽기 지문으로 사용
- 복습 문항으로 재활용
```

## 6. 반복 노출 매핑

핵심 단어/문형/상황이 7가지 형태로 반복 노출되는 구조.

| 핵심 요소 | 세션 | 단어 | 패턴 | 듣기 | 읽기 | 복습 | 테스트 |
|---|---|---|---|---|---|---|---|
| 어휘: 계산 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 문형: [명사] 주세요 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 상황: 가격 묻기 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 의미: 결제 요청 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

**반복 설계 원칙:**
```
같은 단어 "계산":
- 단어카드: "계산 = thanh toán"
- 퀴즈: "돈을 내는 것은?" → 계산
- 패턴: "카드로 ____할게요" → 계산
- 듣기: "카드로 계산할게요" 들으며 인식
- 읽기: 영수증/가격표에서 계산 관련 내용
- 복습: 오답 시 재출제
- 테스트: TOPIK 형식 문제로 최종 확인
```

## 7. 스크립트 → 대량 생성 프롬프트

```
너는 TOPIK1 한국어 콘텐츠 전문가다.

아래 스크립트를 기반으로 문항을 생성해라.

[스크립트]
A: 물 하나 주세요.
B: 네, 천 원입니다.
A: 카드로 계산할게요.
B: 영수증 드릴까요?
A: 아니요, 괜찮아요.

생성할 것:
1. 핵심 어휘 15개 (한국어 + 베트남어 + 예문)
2. 핵심 패턴 5개 + 변형 예문 3개씩
3. 문장 뱅크 30개 (원문 + 변형 + 확장)
4. 단어 퀴즈 10개 (A/B/C 유형 혼합)
5. 패턴 퀴즈 10개 (A/B/C 유형 혼합)
6. 듣기 문항 10개 (A/B/C 유형 혼합)
7. 읽기 문항 10개 (A/B/C 유형 혼합)
8. 미니 테스트 10개 (TOPIK1 형식)

조건:
- 모든 문항은 스크립트 표현 기반
- TOPIK1 초급 수준
- JSON 형식 출력
```
