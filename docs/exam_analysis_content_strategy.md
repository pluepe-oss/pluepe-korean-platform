## 2. TOPIK I 상세 분석

### 시험 구성
| 영역 | 문항 수 | 시간 | 점수 |
|---|---|---|---|
| 듣기 | 30문항 | 40분 | 100점 |
| 읽기 | 40문항 | 60분 | 100점 |

### 등급 기준
| 등급 | 점수 | 의미 |
|---|---|---|
| 1급 | 80~139점 | 기본 생존 한국어 |
| 2급 | 140~200점 | 일상생활 가능 수준 |

### 문제 유형 분석
| 유형 | 내용 | 콘텐츠 활용 |
|---|---|---|
| 짧은 대화 이해 | 편의점/카페/식당 등 생활 상황 | 유닛 세션 영상 기반 |
| 그림 보고 고르기 | 사물/장소/행동 식별 | 단어 플래시카드 |
| 내용 일치 | 짧은 담화 듣고 정보 파악 | 미니 테스트 |
| 읽기 - 표지판/안내문 | 간판/가격표/공지 읽기 | 읽기 드릴 |
| 빈칸 채우기 | 문맥에 맞는 단어/표현 | 패턴 빈칸 퀴즈 |

### 핵심 어휘 카테고리 (콘텐츠 제작 기준)
```
생활/쇼핑: 편의점, 카페, 식당, 시장, 은행, 약국
교통/위치: 지하철, 버스, 오른쪽, 왼쪽, 똑바로
숫자/가격: 얼마, 원, 개, 봉지, 통, kg
시간/날짜: 몇 시, 월, 일, 아침, 저녁
상태/감정: 아파요, 힘들어요, 바빠요, 좋아요
```

## 3. TOPIK II 상세 분석

### 시험 구성
| 영역 | 문항 수 | 시간 | 점수 |
|---|---|---|---|
| 듣기 | 50문항 | 60분 | 100점 |
| 쓰기 | 4문항 | 50분 | 100점 |
| 읽기 | 50문항 | 70분 | 100점 |

### 등급 기준
| 등급 | 점수 | 의미 |
|---|---|---|
| 3급 | 120~149점 | 일상 의사소통 |
| 4급 | 150~189점 | 대학 진학 기준 |
| 5급 | 190~229점 | 전문 분야 가능 |
| 6급 | 230~300점 | 원어민 수준 |

### 쓰기 영역 구조 (핵심 차별화)
| 유형 | 내용 |
|---|---|
| 문장 완성 | 단답형 |
| 설명문 | 200~300자 |
| 논술형 | 600~700자 |

### TOPIK II 유닛 구조 (TOPIK I과 차이점)
| 단계 | TOPIK I | TOPIK II |
|---|---|---|
| 세션 | 짧은 대화 영상 | 지문+대화 혼합 |
| 단어 | 생활 기본 단어 8개 | 추상/확장 단어 10~12개 |
| 패턴 | 짧은 반복 문장 | 연결/복합 문장 (그래서, 하지만) |
| 테스트 | 단순 4지선다 | 독해+문맥 기반 |
| AI | 상황 변형 | 쓰기 조합형 + 짧은 입력형 |

### 쓰기 AI 연동 방식 (베타 기준)
```
1단계: 조합형 쓰기 (베타 오픈)
- 단어 카드 제공 → 문장 순서 맞추기
- 빈칸 채우기 (보기 제공)
- 자동 채점 가능

2단계: 짧은 입력형 (베타 이후)
- 한 문장 직접 입력
- 핵심 단어 포함 여부 체크
- Claude API 간단 피드백
```

## 4. EPS-TOPIK 상세 분석

### 시험 구성
| 영역 | 문항 수 | 시간 | 점수 |
|---|---|---|---|
| 읽기 | 20문항 | 25~35분 | 100점 |
| 듣기 | 20문항 | 25~35분 | 100점 |
| 합계 | 40문항 | 50~70분 | 200점 |

### 합격 기준
- 약 80점 이상 (상대평가 + 선발 방식)
- 대상: 18~39세 외국인 (E-9 비자 취업 목적)

### 듣기 문항 6가지 유형 (핵심)

| 유형 | 문항 | 내용 | 콘텐츠 활용 |
|---|---|---|---|
| 유형1: 음성 식별 | 21~24번 | 단어/단문 정확히 듣기 | 단어 듣기 반복 훈련 |
| 유형2: 시각+언어 | 25~29번 | 그림 보고 맞는 표현 고르기 | 그림+단어 매칭 퀴즈 |
| 유형3: 상황 응답 | 30~33번 | 질문에 맞는 대답 고르기 | 대화 패턴 훈련 |
| 유형4: 맥락 연결 | 34~35번 | 이어지는 말 고르기 | 논리 연결 훈련 |
| 유형5: 직무 내러티브 | 36~37번 | 상황 설명+그림 연결 | 현장 상황 유닛 |
| 유형6: 종합 이해 | 38~40번 | 긴 대화 듣고 정보 추출 | 종합 듣기 테스트 |

### EPS 핵심 특징 (콘텐츠 설계 기준)
```
2회 반복 청취 시스템
- 1회차: 핵심 요지(Gist) 파악
- 2회차: 세부 사항(Detail) 확인
→ 플랫폼 UI: 1회/2회 듣기 토글 버튼 필요
```

### EPS 필수 어휘 카테고리

| 분류 | 핵심 어휘 | 문항 유형 |
|---|---|---|
| 장소/교통 | 시청역, 계류장, 과수원, 은행, 공장, 기차, 오토바이 | 유형2,4 |
| 농업/작업 | 배나무, 따다, 포장하다, 배달하다, 채소 | 유형5 |
| 보호구/안전 | 장화, 안전모, 안전벨트 | 유형3 |
| 미각/상태 | 쉽니다(sour), 짭니다, 답니다, ~것 같아요 | 유형2 |
| 근태/생활 | 야근, 조기퇴근, 감기, 몸살, 휴가 | 유형6 |
| 수량/가격 | 한 봉지, 한 통, 한 개, 1kg, 얼마 | 유형1 |

### EPS 패턴 (명령형 고정)
```
기본 패턴:
- 하세요 / 하지 마세요 / 하십시오
- 안전모를 쓰세요
- 여기서 기다리세요
- 기계를 멈추세요
- 조심하세요

상황 응답 패턴:
- 지금 청소할게요
- 일찍 퇴근해도 될까요?
- 열이 나고 몸이 아파요
```

| 번호 | 주제 | 핵심 상황 |
|---|---|---|
| 1 | 공장 작업 기초 | 작업 시작/종료 지시 |
| 2 | 안전 수칙 | 보호구 착용, 위험 신호 |
| 3 | 농업 현장 | 수확, 포장, 배달 |

| 6 | 근태 관리 | 출퇴근, 야근, 조퇴 |
| 7 | 건강/응급 | 부상, 병원, 약국 |
| 8 | 교통/이동 | 대중교통, 길 찾기 |

## 6. 플랫폼 UI/UX 설계 반영 사항

### EPS 전용 UI 필요 기능
```
1. 2회 반복 청취 토글 버튼
   - 1회차 버튼 / 2회차 버튼 분리
   - 1회차 후 2회차 자동 활성화

2. 시간 압박 퀴즈
   - 듣기 후 3초 내 선택 유도
   - 카운트다운 표시

3. 산업군 필터
   - 농업 / 제조 / 서비스 / 건설
   - 직군별 맞춤 학습 경로
```

### TOPIK II 전용 UI 필요 기능
```
1. 쓰기 입력창
   - 조합형: 드래그앤드롭 단어 배열
   - 입력형: textarea + 글자수 카운터

2. 긴 지문 표시
   - 스크롤 가능한 지문 박스
   - 파란색 박스 스타일 (기존 exam 스타일)

3. AI 첨삭 피드백
   - 핵심 단어 포함 여부 표시
   - Claude API 간단 코멘트
```

## 7. 콘텐츠 제작 우선순위

| 시험 | 콘텐츠 | 수량 | 방법 |

| TOPIK I | 유닛 JSON (2~5) | 4개 | Claude Chat |

| TOPIK I | 듣기 음원 | 30개 | OpenAI TTS |

### 주의사항
```
⚠️ EPS 듣기 음원 제작 시
- 2인 대화 형식 (A/B 역할 분리)
- 자연스러운 속도 (빠르지 않게)
- 산업 현장 어휘 발음 명확하게
```
