## 3. 시험별 유닛 구조 차이

### TOPIK1 (현재 완성)
- 특징: 생존형 회화, 짧은 문장, 반복 구조
- 유닛 수: 12개
- 유닛 구조: 세션(영상) → 단어 → 패턴 → 테스트 → AI 확장
- 지원 언어: 베트남어 / 영어 / 중국어 / 인도네시아어

### TOPIK2 (향후 개발)
- 특징: 읽기/쓰기 중심, 긴 문장, 문맥 학습
- 유닛 수: 20~24개 (12개로 부족)
- 유닛 구조: 지문 읽기 → 표현 분석 → 문장 확장 → 문제 풀이 → 쓰기 연습 → AI 첨삭
- 핵심 차이: AI 쓰기 첨삭 기능 필수

### EPS-TOPIK (향후 개발)
- 특징: 산업/현장 중심, 어휘 제한, 반복 강도 높음
- 유닛 수: 15개
- 유닛 구조: 상황(작업/안전) → 단어(핵심 20개) → 패턴(명령형) → 듣기 훈련 → 시험 문제
- 핵심 차이: 산업 현장 카테고리 기준, 듣기 비중 높음

## 4. 28일 학습 흐름 설계

### 전체 Phase
| Phase | 기간 | 내용 |
|---|---|---|
| Phase 1 | Day 1~12 | 유닛 학습 (1일 1유닛) |
| Phase 2 | Day 13~20 | 복습 + 약점 보완 |
| Phase 3 | Day 21~28 | 모의고사 + 실전 |

### 하루 학습 구조 (20~30분)
| 순서 | 활동 | 시간 |
|---|---|---|
| 1 | 단어 | 5분 |
| 2 | 패턴 | 5분 |
| 3 | 세션(영상) | 10분 |
| 4 | 테스트 | 5분 |
| 5 | AI 확장 | 선택 5분 |

### 자동 추천 로직
```
Day 1~12  → 오늘 할 것: 유닛 학습
Day 13~20 → 오늘 할 것: 약점 복습
Day 21~28 → 오늘 할 것: 모의고사
```

## 5. 마이페이지 설계 (학습 엔진)

### 마이페이지 = 학습 컨트롤 타워

| 섹션 | 내용 | 역할 |
|---|---|---|
| Today | 오늘 할 학습 자동 추천 + 이어하기 버튼 | 학습 진입점 |
| Learning Flow | 현재 Phase 위치 표시 (Phase 1/2/3) | 심리적 방향성 |
| Progress | 유닛 완료 수 / 전체, 연속 학습일(streak) | 동기 부여 |
| Weakness | 단어/패턴/듣기/읽기 약점 분석 | 개인화 |
| Exam | 모의고사 현황, 최근 점수 추이 | 시험 대비 |

### 핵심 UX 원칙
```
"오늘 뭐할까요?" ❌ (선택하게 하면 안 됨)
"다음 행동 자동 제시" ✅ (생각하지 않는 학습 경험)
```

### 필요 데이터
```
user_progress   - 유닛/섹션 완료 기록
user_day        - 학습 시작일 기준 현재 Day
weakness_score  - 영역별 약점 점수
exam_score      - 모의고사 점수 이력
last_activity   - 마지막 학습일
```

## 7. 프론트엔드 메뉴 트리

### 공개 영역 (비로그인)
```
/                    랜딩 페이지
/pricing             상품 소개 + 가격
/free-trial          무료 체험 (유닛 1)
/auth                로그인
/auth/signup         회원가입
```

### 학습자 영역 (Student)
```
/my                  마이페이지 (학습 컨트롤 타워)
/unit/[id]           유닛 훈련 플레이어
  └ session          세션 플레이어 (STEP 1~5)
  └ words            단어 학습
  └ patterns         패턴 학습
  └ test             미니 테스트
  └ ai               AI 확장
/courses             강의 목록
  └ /topik1          TOPIK1 유닛 목록
  └ /topik2          TOPIK2 유닛 목록
  └ /eps-topik       EPS-TOPIK 유닛 목록
/exam                시험 허브
  └ /[type]          IBT 모의고사 플레이어
  └ /result/[id]     시험 결과 + 약점 분석
/vocab               단어 암기 (SRS 간격 반복)
```

### Admin 영역 (학원 원장/강사)
```
/admin               대시보드 (학생 수 / 좌석 / 평균 진도)
/admin/students      학생 목록 + 진도 현황
/admin/students/invite 학생 초대
/admin/reports       학습 리포트
```

### Master 영역 (pluepe 운영팀)
```
/master              전체 학원 / 구독 / 매출 요약
/master/academies    학원 관리
/master/courses      강의 등록 + Bunny 업로드
/master/users        전체 사용자 관리
```

## 8. 백엔드 시스템 구조

### 핵심 도메인
| 도메인 | 역할 | 기술 |
|---|---|---|
| 인증 | 로그인/회원가입/역할 관리 | Supabase Auth |
| 콘텐츠 | 유닛 JSON + 영상 서빙 | Bunny.net Stream |
| 진도 | 섹션/유닛 완료 기록 | Supabase user_progress |
| 결제 | 구독 + 무료체험 + 웹훅 | Stripe |
| AI | 확장 연습 + 오답 해설 + 쓰기 첨삭 | Claude API |
| 시험 | 문항 관리 + 채점 + 약점 분석 | Supabase questions |
| 메일 | 체험 만료 알림 | Resend |

### API 라우트
```
/api/ai              Claude API 프록시
/api/progress        유닛 섹션 진도 저장
/api/exam/questions  시험 문항 조회
/api/exam/submit     시험 채점
/api/stripe/checkout 결제 세션 생성
/api/stripe/webhook  결제 이벤트 처리
/api/stripe/portal   구독 관리 포털
/api/admin/invitations 학생 초대
/api/cron/trial-reminders 체험 만료 알림 크론
```

### 핵심 DB 테이블
```
users           프로필 + 역할 (master/admin/student)
academies       학원 조직
courses         강의 (레거시)
user_progress   유닛 섹션 완료 기록 (신규 핵심)
exam_results    시험 응시 기록
questions       시험 문항
subscriptions   Stripe 구독 상태
seats           학원 좌석
```

## 9. 시험 시스템 구조

- 자동 채점

- AI 오답 해설
- 점수 추이 차트
