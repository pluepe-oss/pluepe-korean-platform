## 4. 학습 구조 설계

### 4-1. 콘텐츠 기준 구조

```
[단어 + 패턴] ← 핵심 기준 (먼저 정의)
      ↓
[영상 대본]   ← 상황 적용 예시 (중간 다리)
      ↓
[문제]        ← 시험 변형 (템플릿 기반 생성)
```

### 4-2. 학습 흐름 (Curriculum_Design 기준)

| 순서 | 학습 타입 | 목적 | 시간 | 우선순위 |
|---|---|---|---|---|
| 0 | 온보딩/진단 | 목표와 수준 파악 | 5~10분 | 선택 |
| 1 | 단어 워밍업 | 세션 이해도 확보 | 5분 | MVP |
| 2 | 문장 패턴 드릴 | 표현 틀 자동화 | 5분 | MVP |
| 3~7 | 세션 STEP 1~5 | 맥락 이해 + 표현 체득 | 10분 | MVP |
| 8 | 문법 브리지 | 시험형 문법 연결 | 5분 | MVP+ |
| 9 | 듣기 드릴 | TOPIK I 듣기 대비 | 5~10분 | MVP+ |
| 10 | 읽기 드릴 | TOPIK I 읽기 대비 | 5~10분 | MVP+ |
| 11 | 미니 테스트 | 단원 점검 | 7~12분 | MVP |
| 12 | 오답 복습 | 약점 개선 | 5분 | MVP |
| 15 | 결과 리포트 | 학습 피드백 | 즉시 | MVP+ |

### 4-3. 1유닛 구조 (MVP 기준 5섹션)

```
① 세션 플레이어 (STEP 1~5)   ← 핵심 엔진, 영상 중심
② 단어 학습                   ← 어휘 반복
③ 패턴 학습                   ← 문장 구조 자동화
④ 미니 테스트                 ← TOPIK 형식 점검
⑤ AI 확장                     ← 상황 응용
```

## 8. AI 콘텐츠 생성 전략

### 8-1. AI 활용도

| 영역 | 활용도 | 비고 |
|---|---|---|
| 단어 세트 | ✅ 100% | 수준 필터 후 바로 사용 |
| 문장/패턴 | ✅ 95% | 초급 수준 제한 조건 필수 |
| 영상 대본 | ✅ 90% | 사람 검수 권장 |
| 문법 설명 | ⚠️ 70~80% | 간결한 설명으로 제한 |
| 시험 문제 | ⚠️ 60~70% | 템플릿 고정 + AI가 내용만 채우기 |

### 8-2. 문제 템플릿 라이브러리

```
[듣기 템플릿]
A: [문장]
B: [문장]
문제: 무엇을 [동사]했습니까?
① ② ③ ④

[읽기 템플릿]
[표지판/가격표/안내문]
문제: [무엇]은 얼마/어디입니까?
① ② ③ ④

[상황 이해 템플릿]
[문장]
문제: 이 사람은 무엇을 합니까?
① ② ③ ④
```

> 템플릿 고정 → AI는 내용만 채우기 → 시험 규격 자동 유지
